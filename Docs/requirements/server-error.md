# 서버 5xx 에러 탐지 요구사항

대상 코드: `src/jobs/server-error.job/job.ts`

## 책임 범위

- 도메인별 5xx 응답 증가를 탐지한다.
- 전체 로그 대비 5xx 응답 비율과 5xx 응답 수를 함께 검증한다.
- 설정된 예외 Domain은 탐지 대상에서 제외한다.

## 요구사항

| 이슈 번호 | 요구사항 ID | 구분 | 요구사항명 | 요구사항 설명 | 우선순위 | 난이도 | 비고 |
| ---: | --- | --- | --- | --- | --- | --- | --- |
| #12 | REQ-012 | 탐지 기능 | 서버 5xx 에러 급증 탐지 | 최근 n분 로그를 domain별로 집계하고, 5xx 응답 수가 n회 이상이면서 전체 로그 대비 n% 이상이면 서버 에러 급증으로 판단해야 한다. | P0 | 중 | 장애 탐지 중요도 높음 |
| #13 | REQ-013 | 예외 처리 | 서버 에러 탐지 예외 Domain 적용 | 5xx 서버 에러 급증 탐지 시 config에 등록된 예외 domain은 탐지 대상에서 제외되어야 한다. | P0 | 하 | REQ-012와 함께 구현 |

## 공통 의존 요구사항

- REQ-002: 탐지 시간 범위, 5xx 응답 수 기준, 비율 기준, 예외 Domain 목록은 config에서 관리한다.
- REQ-004: 최근 n분 로그를 Elasticsearch ESQL API로 조회한다.
- REQ-005: `timestamp`, `client_ip`, `domain`, `path`, `protocol_status`를 내부 분석 구조로 파싱한다.
- REQ-018: 알림 본문에 탐지 유형, 대상 domain, 발생 횟수, 오류 비율, 탐지 기준, 탐지 사유가 전달될 수 있어야 한다.
- REQ-026: 탐지 결과 생성과 실패 상황을 로그로 확인할 수 있어야 한다.
- REQ-028: 정상 탐지, 미탐지, 예외 Domain 케이스를 단위 테스트로 검증한다.
